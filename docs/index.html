Code naming conventions.
Class/object property, prefixed with "__" (double underscore), is a public property that is guaranteed to have both a getter and a setter methods with the same clear names without any prefixes or postfixes.
E.g.,

...

constructor() {
  this.__counter = 0;
}

get counter() {return __counter};
set counter(v) {
  typeof v === "number"
  ? this.__counter = v
  : throw new SyntaxError("Counter property can be set to a number only.")}

...
